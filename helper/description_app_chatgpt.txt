
Цель проекта
Это система управления задачами / трекер привычек, с возможностью объединять задачи в блоки,
повторять их по расписанию, использовать пользовательские шаблоны и группы.
Ключевая идея: не просто “список задач”, а динамическая система с повторением, приоритетами и блоками.

Основные сущности

Сущность	Назначение
TaskTemplateUser	Шаблон задачи пользователя: название, описание, изображение, период повторения, расписание, приоритет, количество
GroupTemplateUser	Шаблон группы пользователя: название, описание, список шаблонных задач, количество раз выбора
UserTask	Конкретная задача пользователя, созданная из шаблона (может хранить выполнено/не выполнено, last_completed_at)
Block	Объединение задач пользователя, можно рассматривать как “день / набор задач”
BlockTask	Связь блок → задача

Типы повторений
    none — однократная задача
    week / month / year — периодические задачи
schedule_type:
    fixed — задача привязана к дню недели или дню месяца
    floating — выполнить в любой день периода
    none — нет расписания

Логика активации задач
    Задача активна, если она ещё не выполнена в текущем периоде
    Для “floating week/month” проверяем, что текущая неделя/месяц > last_completed_at
    Для “fixed” проверяем совпадение с днем/днем недели + период
    Активация считается на лету при показе задач, нет отдельного поля is_active (кэш можно добавлять для оптимизации)

Блоки
    Блоки — это объединение пользовательских задач, например “Сегодня”, или пользовательский набор из группы
    Создаются путем копирования задач из группы шаблонов
    Каждый блок содержит свои UserTask, выполнение которых не влияет на другие блоки (каждая копия независима)

Приоритеты и сортировка
Активные задачи → впереди
Неактивные → после

Внутри сортировка по приоритету → затем дата создания
Пример жизненного цикла
Пользователь создаёт блок или выбирает группу

Система копирует задачи из шаблонов → создаёт UserTask → связывает с блоком

Пользователь видит задачи, активные идут первыми

Пользователь выполняет задачу → last_completed_at обновляется

Система пересчитывает активность для следующего периода
